wiki

Download an Azure DevOps wiki to your local filesystem.

Setup

  1. Create a PAT in Azure DevOps with Wiki (Read) and Code (Read) scopes. Wiki scope pulls page content; Code scope is required to download image attachments from the wiki's backing git repo.
  2. Make a folder for your wiki, e.g. ~/my-wiki.
  3. Download the right archive from the latest release, extract it, and drop the wiki binary (wiki.exe on Windows) into that folder:
    • Windows — wiki_*_windows_amd64.zip (Intel/AMD) or wiki_*_windows_arm64.zip
    • macOS — wiki_*_darwin_arm64.tar.gz (Apple Silicon: M1/M2/M3/M4) or wiki_*_darwin_amd64.tar.gz (Intel Macs)
    • Linux — wiki_*_linux_amd64.tar.gz or wiki_*_linux_arm64.tar.gz
  4. From that folder, generate a config template: 
    cd ~/my-wiki
./wiki sync          # writes wiki.yaml, then exits
  5. Edit wiki.yaml: 
    organization: your-azure-devops-organization
project: Your Project Name
wiki: Your Project.wiki

Use

cd ~/my-wiki
./wiki login         # store PAT
./wiki sync          # pulls pages + attachments into articles/<wiki>/
./wiki logout        # remove PAT

Images land in articles/<wiki>/.attachments/ so /.attachments/… references in the markdown resolve in any viewer.

Limitations

  • Cross-page links don't resolve when opening .md files directly.
  • Local edits to synced files are lost if the page is deleted upstream.
